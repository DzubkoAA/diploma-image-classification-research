# Разработка и исследование системы классификации транспортных средств на основе сверточных нейронных сетей

## О проекте

Исследование посвящено решению задачи классификации транспортных средств на изображениях. Основная цель — поиск оптимального баланса между точностью распознавания, скоростью работы и требованиями к вычислительным ресурсам для последующего развертывания на мобильном устройстве.

В работе проведен комплексный анализ от выбора архитектуры нейронной сети до создания рабочего Android-приложения.

---

## Исследованные архитектуры

| Архитектура | Ключевая идея | Особенности |
|-------------|---------------|-------------|
| **ResNet** | Остаточные связи (skip connections) | Решает проблему затухания градиентов, стабильное обучение |
| **EfficientNet** | Компаундное масштабирование | Оптимальный баланс точности и затрат |
| **MobileNetV2** | Depthwise separable свертки | Создана для мобильных устройств, малый вес |
| **ConvNeXt** | Современная модернизация CNN | Высокая точность на больших данных |
| **SqueezeNet** | Fire модули | Экстремально компактная |

## Результаты экспериментов

| Модель | Accuracy | Основное преимущество | Недостаток для нашей задачи |
|--------|----------|------------------------|------------------------------|
| **MobileNetV2** | 0.922 | Оптимальный баланс точности и скорости | Небольшое отставание от лидеров |
| **ResNet-50** | 0.926 | Высокая стабильность | Больший размер модели |
| **EfficientNet-B0** | 0.938 | Лучшая точность | Более сложная интеграция |
| **ConvNeXt-Tiny** | 0.835 | Современная архитектура | Значительно ниже точность |
| **SqueezeNet** | 0.716 | Компактность | Недостаточная точность |

### Анализ

- **EfficientNet-B0** показал лучшую точность (0.938), но сложнее в интеграции
- **ResNet-50** продемонстрировал стабильность (0.926) при большем размере
- **MobileNetV2** выбран как оптимальный вариант (0.922) благодаря балансу точности, скорости и компактности
- **ConvNeXt-Tiny** и **SqueezeNet** показали недостаточную точность для практического применения

Наиболее частые ошибки моделей — путаница между визуально похожими классами (автобусы/грузовики).

## Мобильное приложение

На основе результатов для интеграции выбрана **MobileNetV2** как оптимальная модель.

### Функционал:
- Съемка фото непосредственно в приложении
- Загрузка изображений из галереи
- Мгновенное распознавание с отображением класса и уверенности модели
- Сохранение истории распознаваний

### Тестирование
- Приложение работает стабильно на реальных устройствах
- Скорость обработки достаточна для комфортного использования
- Ошибки носят ожидаемый характер и связаны с объективными сложностями (ракурс, тень, сходство классов)
